\begin{abstract}
Laporan ilmiah ini menyajikan perancangan dan implementasi dari \textit{"Adaptive Safety-Critical Multi-Sensor Acquisition Framework"} berbasis STM32F401VE di bawah sistem operasi \textit{real-time} FreeRTOS. Pengembangan framework ini didasarkan pada sintesis komprehensif arah riset masa depan (\textit{future work}) dari 16 artikel jurnal internasional bereputasi (Scopus/WoS Q1-Q3) dalam rentang tahun 2021--2026. Kerangka kerja yang diusulkan mengintegrasikan empat inovasi arsitektur utama: akuisisi data multi-channel ADC berbasis DMA dengan laju \textit{sampling} adaptif, penjadwalan hibrida prioritas-deadline pada FreeRTOS, penempatan buffer kritis pada area \textit{Core Coupled Memory} (CCM) untuk meminimalisasi latensi, serta sistem monitoring keselamatan berbasis \textit{Control Barrier Function} (CBF) terintegrasi dengan Window Watchdog (WWDG). Hasil pengujian fungsional pada simulator Proteus 8 membuktikan efisiensi komputasi, keandalan kontrol aktuator PWM 10 kHz, serta kepatuhan penuh terhadap batas batas keselamatan yang ditetapkan (\textit{fault barrier}), dengan pemutusan otomatis (\textit{fail-safe}) saat mendeteksi bahaya sistemik.
\end{abstract}

\subsection{Dasar Penggabungan Future Work}
Framework ini dirancang untuk mengatasi inefisiensi pada sistem kontrol konvensional yang membaca sensor pada frekuensi tetap (\textit{fixed rate}), tidak memiliki pemisahan memori yang jelas antara data rutin dan kritis, serta hanya mengandalkan \textit{watchdog reset} pasif tanpa proteksi barrier aktif. Penggabungan taktik ini direalisasikan dalam 4 komponen dasar:
\begin{enumerate}
	\item \textbf{Laju Sampling Adaptif (ADC DMA):} Memanfaatkan deteksi dinamis level sinyal analog. Jika perbedaan nilai sensor di bawah batas ambang (\textit{quiescent state}), delay sampling ditingkatkan menjadi 100 ms untuk menghemat daya (clock-gating logis). Jika sinyal berubah cepat (\textit{active state}), delay dipercepat hingga 10 ms demi akurasi tinggi.
	\item \textbf{SRAM Core-Coupled Memory (CCM):} Seluruh buffer DMA (\texttt{adc\_dma\_buf}) diletakkan di memori akses cepat untuk mencegah interupsi dari bus utama mikrokontroler.
	\item \textbf{Priority-Deadline Hybrid Scheduler:} FreeRTOS mengelola 4 task terpisah:
	\begin{itemize}
		\item \texttt{Task\_SafetyCheck}: Prioritas \textit{Realtime} (Prio 4) - Berjalan setiap 5 ms untuk validasi CBF.
		\item \texttt{Task\_SensorRead}: Prioritas \textit{High} (Prio 3) - Berjalan adaptif (10--100 ms) untuk akuisisi ADC.
		\item \texttt{Task\_DataProcess}: Prioritas \textit{Normal} (Prio 2) - Eksekusi filter \textit{moving average} dan kalkulasi PWM.
		\item \texttt{Task\_CommOutput}: Prioritas \textit{Low} (Prio 1) - Transmisi data serial via UART dan I2C.
	\end{itemize}
	\item \textbf{Control Barrier Function (CBF):} Menyediakan barrier numerik (200 mV--2800 mV). Jika sensor melanggar batas berturut-turut sebanyak 3 kali, sistem beralih ke state \texttt{FAULT}, memutus daya aktuator (PWM 0\%), menyalakan indikasi bahaya, dan menolak mereset WWDG agar sistem melakukan \textit{hard reboot} untuk proteksi.
\end{enumerate}

\subsection{Analisis Perilaku Sistem}
Berdasarkan data dan grafik pengujian:
\begin{itemize}
	\item \textbf{Fase Awal (0--1500 ms):} Dinamika sensor sangat lambat (perubahan $< 50$ mV per iterasi). Sistem secara otomatis mengaktifkan modus hemat daya dengan memperlebar inter-sampling delay ke 100 ms.
	\item \textbf{Fase Dinamis (2000--10000 ms):} Pengguna memutar potensiometer secara drastis. Perbedaan tegangan melompati ambang batas deviasi ($> 100$ mV). Sistem merespons dengan meningkatkan frekuensi akuisisi (delay = 10 ms). Output PWM TIM1 beraksi linier dan proporsional untuk meminimalkan keterlambatan kontrol.
	\item \textbf{Fase Bahaya (17000--18200 ms):} Input sensor 1 diputar ekstrim mendekati rel suplai (VCC). Pada 17400 ms, tegangan menembus batas CBF ($> 2600$ mV). State berubah menjadi \texttt{WARNING} (LED oranye menyala). Pada 18000 ms, Sensor 1 terbaca 2850 mV (melewati batas kritis atas 2800 mV) selama 3 kali siklus berturut-turut. Sistem langsung memasuki fase \texttt{FAULT} (LED merah aktif), dan PWM diturunkan seketika ke 0\% (\textit{fail-safe state}) untuk menyelamatkan perangkat fisik dari panas berlebih atau kerusakan beban.
\end{itemize}

\section{Kesimpulan}
Framework akuisisi \textit{"Adaptive Safety-Critical Multi-Sensor Framework"} telah berhasil dirancang, disimulasikan, dan diuji fungsionalitasnya secara komprehensif. Melalui sintesis 16 penelitian terdahulu, framework ini membuktikan bahwa integrasi taktik akuisisi adaptif, penjadwalan berorientasi deadline keselamatan, penempatan memori performa tinggi, dan barrier proteksi aktif mampu mewujudkan sistem instrumentasi yang tangguh, hemat daya, dan aman. Pengujian fungsional Proteus memvalidasi keandalan respon PWM 10 kHz dan fungsi \textit{fail-safe} pemutusan otomatis yang responsif saat sensor melanggar batas keselamatan. Framework ini sangat potensial untuk diaplikasikan langsung pada sistem otomotif, kedokteran, maupun industri manufaktur berbasis mikrokontroler fisik.
